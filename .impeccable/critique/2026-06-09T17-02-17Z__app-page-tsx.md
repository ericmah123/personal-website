## Priority Issues

### [P3] Bug: clicking the "menu" label doesn't open the nav
nav-pill-hint span is outside the button element. Clicking "menu" hits the e.target !== e.currentTarget guard and does nothing. Visible only to reduced-motion users who already opted out of animation — worse than useless if it looks broken.

**Fix:** Move span inside the button element. CSS selector .nav-pill:not(.nav-open) .nav-pill-hint still resolves correctly.

### [P3] No skip link for keyboard users
Users Tab through nav toggle and four links before reaching main content. A visually-hidden skip link (#main-content) would complete the keyboard navigation story.

### [P3] Flip card current face has no indicator
Rotate icon communicates "flippable" but not "you're on the back face." Low stakes on a portfolio but could add confusion if users scroll away mid-flip.
